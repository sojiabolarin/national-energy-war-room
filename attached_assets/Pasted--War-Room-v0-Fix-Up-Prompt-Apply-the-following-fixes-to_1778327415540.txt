## Fix 1 — Plant marker colour-coding

**The bug:** All generation plant markers on the map render as translucent grey circles. The legend promises Operating=green, Partial=amber, Constrained=orange, Out=red. The status colour is not being applied to the marker fill.

**Fix:** In the Leaflet map component, when rendering each plant's `circleMarker`, set `fillColor` based on `plant.status`:
- `OPERATING` → `#2E7D32` (green)
- `PARTIAL` → `#ED6C02` (amber)
- `CONSTRAINED` → `#E85426` (brand orange — this matches the legend's "Constrained" swatch)
- `OUT` / `MAINTENANCE` → `#C62828` (red)

Set `fillOpacity: 0.85`, `color: 'white'` (border), `weight: 1.5`. Marker radius scales with `installedMw` using `6 + Math.sqrt(installedMw) / 8` (caps at ~25px for the largest plants).

**Acceptance:** Open Generation Map. Egbin (Lagos, OPERATING at ~46% PAF) renders green; Olorunsogo NIPP (gas-constrained) renders orange; Alaoji NIPP (zero output Sep 2025) renders red. At least four distinct colours visible across plant markers.

## Fix 3 — Live state indicator confusion

**The bug:** The map shows "CONNECTING…" at the top-centre while simultaneously the panel header reads "LIVE · Press F for fullscreen". Two conflicting signals about live state.

**Fix:** Single source of truth. The SSE client should track `connectionState: 'connecting' | 'live' | 'reconnecting' | 'offline'` and render one indicator only, in the panel header.

Render rules:
- `connecting` (initial only): pulse animation, label "Connecting…"
- `live`: solid green dot, label "Live · {timestamp of last message}"
- `reconnecting`: amber pulse, label "Reconnecting…"
- `offline`: red, label "Offline · last update {time}"

Remove the centre-of-map "CONNECTING…" overlay entirely. If SSE never connects in dev (because the worker isn't running), label honestly as "Offline" rather than perpetually "Connecting". Add a `?mock=1` query param the agent can use to inject simulated frequency updates for demos when the live worker isn't available — and in that mode the indicator shows "Demo mode · simulated data" in amber, never "Live".

**Acceptance:** No "CONNECTING…" overlay on the map. Header shows exactly one indicator that reflects the actual connection state. Killing the API server changes the indicator to "Offline" within 5 seconds.
