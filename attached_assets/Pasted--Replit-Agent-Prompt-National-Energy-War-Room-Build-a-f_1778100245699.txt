# Replit Agent Prompt — National Energy War Room

**Build a full-stack platform consisting of (1) a public-facing operational dashboard, (2) an admin backend for the data managers, and (3) a REST API layer. Add a first-class Customer Complaints & Escalation module that runs through all three.**

This is a procurement-grade system for the Federal Ministry of Power, Federal Republic of Nigeria. The product is the National Energy War Room. Build it for production reliability, not as a demo. Pay close attention to data integrity, role-based access, and audit trails — every action by an admin must be logged and attributable.

---

## 1. Stack & Project Setup

- **Frontend:** React 18 with Vite, React Router v6, TanStack Query for server state, Zustand for local state, Recharts for charts, Leaflet (`react-leaflet`) for maps.
- **Backend:** Node.js 20 + Express, Prisma ORM, PostgreSQL, Zod for request validation, jsonwebtoken for auth, bcrypt for password hashing, pino for structured logging.
- **Real-time:** Server-Sent Events (SSE) for one-direction live updates (frequency, alerts, complaint counts). Do not use WebSockets unless requirements expand.
- **Brand:** WestMetro Limited brand standard. **Arial only.** Colors: orange `#E85426`, dark `#1A1A1A`, body `#1C1C1C`, gray `#F4F4F4`. No navy, no gold, no other accent colors. Dark theme by default.
- **Project layout:** Monorepo with `apps/web` (React frontend), `apps/admin` (admin frontend, separate React app), `apps/api` (Express backend), `packages/shared` (TypeScript types and Zod schemas). Use `pnpm` workspaces.
- **Environment:** `.env` files per app. `DATABASE_URL`, `JWT_SECRET`, `JWT_EXPIRY`, `PORT`, `CORS_ORIGINS`.

Set up Replit so all three apps run concurrently in dev. Use `concurrently` package.

## 2. Data Model (Prisma schema)

Define these entities. Use UUIDs for all IDs. Add `createdAt`, `updatedAt`, `createdBy`, `updatedBy` audit fields to every mutable entity.

### Core entities

- **User** — id, email (unique), passwordHash, role (enum: MINISTER, MINISTRY_STAFF, NERC_VIEWER, DISCO_AGENT, CITIZEN, ADMIN), fullName, organisationId (nullable), phone, createdAt, lastLoginAt, isActive.
- **Organisation** — id, name, type (enum: GENCO, DISCO, TCN, NBET, NERC, NMDPRA, NGIC, REA, FMP, FGN_POWER_CO, NDPHC, OTHER), parentId (nullable, for divisions), website, createdAt.
- **AuditLog** — id, userId, action (string), entity (string), entityId, before (JSONB), after (JSONB), ip, userAgent, createdAt. Auto-write on every mutation through middleware.

### Power-sector entities

- **Plant** — id, name, type (enum: GAS_OCGT, GAS_CCGT, GAS_STEAM, HYDRO, SOLAR, WIND, COAL, IPP, NIPP), installedMw (decimal), availableMw (decimal), actualMw (decimal), state, latitude, longitude, status (enum: OPERATING, PARTIAL, MAINTENANCE, OUT, CONSTRAINED), gencoOrgId (FK Organisation), commissioningDate, paf (decimal), notes, createdAt, updatedAt. One-to-many with PlantUnit.
- **PlantUnit** — id, plantId, unitName, capacityMw, currentOutputMw, status, fuelStatus, lastTripAt, lastTripCause, notes.
- **TransmissionLine** — id, name, voltageKv (enum: 330, 132), lengthKm, capacityMva, fromSubstationId, toSubstationId, status, currentLoadingPct, lossesPct, notes, geometry (JSONB array of [lat,lng] pairs).
- **Substation** — id, name, voltageClass, capacityMva, transformerConfig, circuitCount, latitude, longitude, state, ownerOrgId.
- **DisCo** — id, name (unique, e.g. "Eko"), licenceNumber, allocationPct, operatorOrgId, registeredOffice, latitude, longitude, atccLossPct, collectionEffPct, billingEffPct, meteringRatePct, hoursOfSupplyDaily, complaintsLastQuarter, rank, badge (enum: GOOD, WARN, BAD), enforcementStatus, mytoTargetAtcc.
- **Feeder** — id, discoId, name, voltage, customerCount, supplyHours, energyBilledMm, atccLossPct, profile, securityRisk (boolean), notes.
- **GasPipeline** — id, name, capacity, operator, status (enum: OPERATIONAL, CONSTRUCTION, PROPOSED), geometry (JSONB), notes, fromPoint, toPoint.
- **DiversionOpportunity** — id, plantId, tapPoint, tapLatitude, tapLongitude, lengthKm, capacityRequired, indicativeCapex, note, priority (1-5), createdBy.
- **CapitalProject** — id, name, category (enum: GENERATION, TRANSMISSION, GAS_INFRA, DRE, DISTRIBUTION, OTHER), status, sponsorOrgId, latitude, longitude, when (text, free-form timeline), capexUsd, completionPct, notes, sourceRef (URL or doc reference).
- **MiniGrid** — id, name, capacityKw, beneficiaries, latitude, longitude, operatorOrgId, programme (enum: NEP, DARES, OTHER), status.

### Settlement entities

- **SettlementInvoice** — id, period (e.g. "2025-Q1" or "2025-04"), discoId, mooInvoiceNgn (decimal), nbetInvoiceNgn, drogAdjustedNgn, remittedNgn, remittancePct, cumulativeDebtNgn, enforcementStatus, notes.
- **GencoAllocation** — id, settlementInvoiceId, gencoOrgId, plantId (nullable), shareNgn, paymentStatus (enum: PAID, PARTIAL, UNPAID, DEFAULT).

### Operational entities

- **GridMetric** — id, timestamp, frequencyHz (decimal), upperVoltageKv, lowerVoltageKv, sentOutMwh, demandMwh. Time-series, indexed on timestamp.
- **Alert** — id, severity (enum: CRITICAL, HIGH, MEDIUM, INFO), category, title, message, sourceEntity, sourceEntityId, actionRequired, status (enum: OPEN, ACKNOWLEDGED, RESOLVED), assignedToUserId, resolvedAt, createdAt.

### Customer Complaints & Escalation

This is the **Customer Complaints & Escalation Module**. It must be a first-class subsystem, not an afterthought.

- **Complaint** — id, ticketNumber (auto-generated, format `WR-YYYYMMDD-NNNNNN`), source (enum: WEB, WHATSAPP, NERC_PORTAL, FORUM_OFFICE, IN_PERSON, EMAIL, IMPORT), citizenName, citizenPhone, citizenEmail, citizenNin (nullable), discoId, feederId (nullable), category (enum: METERING, BILLING, ESTIMATED_BILLING, SUPPLY_INTERRUPTION, VOLTAGE, ELECTROCUTION, INFRASTRUCTURE_DAMAGE, CONNECTION_DELAY, DISCONNECTION, REFUND, ENERGY_THEFT_REPORT, OTHER), subCategory (string), description (text), location (text — address), latitude (nullable), longitude (nullable), severity (enum: LOW, MEDIUM, HIGH, CRITICAL), status (enum: FILED, TRIAGED, ASSIGNED, IN_PROGRESS, AWAITING_CITIZEN, ESCALATED, RESOLVED, CLOSED, REJECTED), escalationLevel (1-5, where 1 = DisCo CCU, 2 = DisCo MD, 3 = NERC CCU, 4 = NERC Forum Office, 5 = Ministerial), escalationReason, slaDueAt (computed from CPR 2023 windows), resolvedAt, resolvedBy, resolution (text), satisfactionScore (1-5, nullable), satisfactionFeedback, attachments (JSONB array), createdAt, updatedAt.
- **ComplaintEvent** — id, complaintId, eventType (enum: CREATED, STATUS_CHANGE, ASSIGNED, ESCALATED, NOTE_ADDED, ATTACHMENT_ADDED, SLA_BREACHED, RESOLVED, REOPENED, CITIZEN_RESPONDED), fromValue, toValue, actorUserId (nullable for system events), notes, createdAt. Append-only; full audit trail.
- **ComplaintAssignment** — id, complaintId, assignedToUserId, assignedFromUserId, assignedAt, dueAt, completedAt, notes.
- **EscalationRule** — id, name, condition (JSONB — DSL describing when this rule fires; e.g. `{ "category": "ESTIMATED_BILLING", "ageHours": 72, "currentLevel": 1 }`), actionLevel (target escalationLevel), notifyOrgId, notifyRole, isActive, priority. Evaluated by a background worker every 5 minutes.

### Stakeholder & Accountability

- **ValueChainLink** — id, key (unique slug e.g. "gas", "gen", "tx"), name, status (G/A/R), meta (rich text/HTML), order. Eight links default: Gas Supply, Generation, Transmission, Distribution, Settlement, Customer & Metering, Off-Grid & DRE, Capital Projects.
- **Stakeholder** — id, valueChainLinkId, role (enum: OPERATOR, REGULATOR, COUNTERPART, FUNDING_PARTNER, AUTHORITY), title (e.g. "MD/CEO, TCN") — **store role titles only, never personal names**. Names change with appointments and deprecate the data instantly. organisationId, description, escalationOrder.
- **AuthorityInstrument** — id, valueChainLinkId, name (e.g. "Generation Licence", "MYTO 2025"), citation, description.
- **EscalationStep** — id, valueChainLinkId, stepOrder, who (role title), whatRole, instrument, notes.

### Notifications

- **Notification** — id, userId, kind, title, body, payload (JSONB), readAt, createdAt.

Run `prisma migrate dev` to generate. Seed with realistic Nigerian Q1 2025 NERC data (see Section 7).

## 3. API Layer (`apps/api`)

REST + JSON, versioned at `/api/v1`. Express + Prisma + Zod. Every route validated through Zod schemas defined in `packages/shared`.

### Conventions

- All errors return `{ error: { code, message, details? } }` with appropriate HTTP status.
- All list endpoints support `?page=`, `?pageSize=` (max 100), `?sort=`, `?filter[field]=value`. Return `{ data: [], pagination: { page, pageSize, total, totalPages } }`.
- All mutations require auth and write an AuditLog row.
- Idempotency: accept `Idempotency-Key` header on POSTs that create resources.
- Rate limit: 100 req/min per IP for unauthenticated, 600 req/min for authenticated, 60 req/min for citizen complaints submission.

### Auth

- `POST /api/v1/auth/register` — citizens only (other roles created by admin). Body: email, phone, password, fullName, nin (optional). Returns user + JWT.
- `POST /api/v1/auth/login` — email + password → JWT.
- `POST /api/v1/auth/refresh` — refresh token.
- `POST /api/v1/auth/logout` — invalidates refresh token.
- `GET /api/v1/auth/me` — current user.
- `POST /api/v1/auth/forgot-password` — sends reset link (stub email service).
- `POST /api/v1/auth/reset-password` — completes reset.

### Access tiers (three explicit tiers — apply consistently across all endpoints)

The API has exactly three access tiers. Every endpoint must declare which tier it belongs to.

**Tier 1 — Public, unauthenticated.** Only the citizen complaint flow lives here. Rate-limit aggressively (60 req/min per IP). All other public-facing endpoints removed from this tier.
- `POST /api/v1/auth/register` (citizen self-registration only — sets role=CITIZEN)
- `POST /api/v1/auth/login`
- `POST /api/v1/auth/refresh`
- `POST /api/v1/auth/forgot-password`
- `POST /api/v1/auth/reset-password`
- `POST /api/v1/complaints` (file a complaint — captures phone for verification)
- `GET /api/v1/complaints/track/:ticketNumber` (requires phone last-4 verification)
- `POST /api/v1/complaints/:id/citizen-response`
- `POST /api/v1/complaints/:id/satisfaction` (token-protected one-tap link)
- `POST /api/v1/complaints/whatsapp/webhook` (Meta signature verification)
- `GET /healthz`

**Tier 2 — Authenticated staff (any non-CITIZEN role).** All sector dashboard data is here. The war room is RESTRICTED — staff only.
- `GET /api/v1/sector/kpis`
- `GET /api/v1/sector/grid/live` and SSE stream
- `GET /api/v1/sector/sankey`
- `GET /api/v1/plants`, `/:id`, `/:id/units`
- `GET /api/v1/transmission/lines`, `/:id`, `/substations`
- `GET /api/v1/discos`, `/:id`, `/:id/feeders`
- `GET /api/v1/gas/pipelines`, `/diversions`
- `GET /api/v1/projects`
- `GET /api/v1/minigrids`
- `GET /api/v1/settlement/invoices?period=`
- `GET /api/v1/value-chain`, `/:key`
- `GET /api/v1/alerts/active` and SSE stream
- `GET /api/v1/auth/me`
- `POST /api/v1/auth/logout`

**Tier 3 — Authenticated staff with elevated role.** Mutations and admin views, role-gated per route:
- `MINISTRY_STAFF`+ — read all complaints, all registry CRUD, settlement entry, alerts CRUD.
- `NERC_VIEWER` — read all complaints + read all registry. No mutations.
- `DISCO_AGENT` — read & update only complaints scoped to their `organisationId` (DisCo). No registry access.
- `ADMIN` — full system, including user management, audit log, escalation rules, system settings.

Every Tier 3 endpoint validates role per request via middleware and writes an AuditLog row.

If a citizen-role user (CITIZEN) hits a Tier 2 or Tier 3 endpoint, return 403 with `{ error: { code: "STAFF_ONLY" } }`.

The dashboard frontend (`apps/web`) and admin frontend (`apps/admin`) both require login before rendering any non-public route. There are NO unauthenticated routes in the dashboard frontend other than `/complaints/file`, `/complaints/track`, `/complaints/satisfaction/:token`, and `/login`.

### Background workers

- **Escalation worker** — every 5 min, evaluate active EscalationRule entries against open Complaints. Auto-escalate when SLA windows breach. Write ComplaintEvent. Send Notification to next-level role.
- **SLA tracker** — every minute, compute hours-since-filed for OPEN complaints. Mark `slaBreached` flag and emit alert when category-specific window passes (from CPR 2023: metering 5 working days, billing 15 working days, supply interruption 24 hours, electrocution immediate).
- **Stats refresher** — every 5 min, materialise complaint stats into a fast-lookup table.

## 7. Seed Data

Seed the database with realistic Nigerian sector data from NERC's Q1 2025 Quarterly Report (publicly published July 2025).

### KPIs (from NERC Q1'25)
- Average available generation: 5,366.88 MW
- Average hourly generation: 4,770.59 MWh/h
- Total quarterly generation: 10,304.47 GWh
- Weighted ATC&C loss: 39.61% (vs MYTO target 20.54%)

## 8. Security & Operations

- **Auth:** JWT signed with RS256 (key in env). 15-min access token, 7-day refresh token. Rotate refresh tokens on each use.
- **CORS:** explicit origin allow-list per environment.
- **Helmet** — set on Express. CSP enabled.
- **Rate limit** — `express-rate-limit` with Redis store in production (memory in dev).
- **Input validation** — every route Zod-validated.
- **Audit** — every mutation writes AuditLog (user, before, after, IP, ua, timestamp).
- **PII** — citizen phone, email, NIN treated as sensitive. Redact in logs. Encrypt NIN at rest (AES-256, key from env).
- **Secrets** — never in code. `.env` only; production via Replit Secrets manager.
- **Backups** — Postgres logical dump nightly to object storage (stub; implement as scheduled job).
- **Health endpoint** — `GET /healthz` returns DB connectivity + worker status.
